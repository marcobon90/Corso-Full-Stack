Guida a Bootstrap

Molti web template utilizzano Bootstrap, un framework front-end che fornisce una serie di strumenti e stili CSS che ci aiutano a creare siti e applicazioni web in modo rapido: basta assegnare agli elementi una classe con il nome della caratteristica che vogliamo dargli.

Tuttavia, l’uso di Bootstrap o framework simili presentano un problema: richiede il caricamento di tutti gli stili anche di quelli che non usiamo nel codice HTML e questo penalizzerà il nostro sito.

La guida seguente riguardo Bootstrap è principalmente dimostrativa e ci permette di capire come realizzare qualcosa di simile. Ma se si desidera creare siti professionali, e con il minore numero di segnalazioni da parte del browser e dei motori di ricerca il suo uso è sconsigliato per i motivi detti sopra.

Installazione

Possiamo installare Bootstrap in due modi: tramite CDN o salvare i file nel progetto In questa guida sto usando il CDN.

Tramite CDN

E’ il metodo più veloce e leggero, in quanto i file di Bootstrap sono distribuiti su server esterni usati anche da altri siti web. Se l’utente ha già visitato un sito che usa il nostro stesso framework avrà già i file nella cache, memoria temporanea, del browser, e i tempi di caricamento saranno minori. Per aggiornare Boostrap a una nuova versione basta modificare l’url della CDN.

Gli svantaggi di usare una CDN è che nel caso ci siano problemi, il sito potrebbe non funzionare correttamente; inoltre, abbiamo meno controllo sui file, nel caso volessimo fare delle modifiche personalizzate.

Installazione locale

Il vantaggio di avere tutti i file di Bootstrap nel nostro progetto è che possiamo modificare i file a seconda delle nostre preferenze. Tuttavia, dovremo scaricare manualmente le nuove versioni che sovrascriveranno le modifiche effettuate. Inoltre, i file devono essere scaricati ogni volta dal nostro server ai vari utenti.

Ha senso avere tutti i file di Bootstrap nel nostro progetto quando vogliamo gestirlo tramite SASS. Grazie al preprocessore possiamo importare soltanto le parti di Bootstrap che ci interessano, modificare le sue variabili e creare funzioni personalizzate senza dovere modificare il suo codice sorgente. In questo modo il nostro sito non sarà appesantito dal codice di stile che non utilizziamo perché incorporeremo nella pagina web soltanto il CSS compilato da SASS.
